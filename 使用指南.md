# 全制程NG不良智能归因分析系统 - 使用指南

## 1. 系统简介
本系统是一个用于分析全制程数据中导致NG（不良品）主要因素的Web应用程序。通过输入制程数据（如几十万条包含数百个参数的记录），系统能够自动进行统计分析，找出与不良品相关性最高的Top N个核心特征（如特定机台、特定参数、特定时间段等），并以直观的图表形式展示。

### 核心算法说明：
在处理“Pass/Fail”这样的二分类目标变量时，系统会自动区分数据类型：
- **分类变量（Categorical）**：如机台ID (MC_ID)、喷嘴 (Nozzle)、批次号 (Lot ID) 等。系统使用 **Cramér's V（基于卡方检验）** 来计算相关性。
- **连续变量（Numerical）**：如时间参数、驻留时间 (Staging_time) 等。系统使用 **点双列相关系数 (Point-biserial correlation)** 来计算相关性。
- 此方案比单纯计算 R² 更契合离散的制程归因场景。

## 2. 环境配置与启动

### 2.1 安装依赖
推荐使用Conda或Python的venv虚拟环境。在项目根目录下运行以下命令安装所需依赖：
```bash
pip install flask pandas numpy scipy openpyxl
```

### 2.2 启动服务
在项目根目录下运行主程序：
```bash
python app.py
```
服务启动后，在浏览器中访问：`http://127.0.0.1:8080/`

## 3. 功能操作说明

### 3.1 数据分析
当你拿到真实的全制程Excel数据时：

1. **上传全制程主表**：
   - 将包含全制程数据的Excel或CSV文件拖拽到左侧的 **“数据输入”** 虚线框内。
   - **注意**：数据中必须包含一列用于判定好坏的列（系统会自动识别名为 `Results` 或内容包含 `Pass/Fail`, `OK/NG` 的列）。

2. **上传筛选列表（可选，但强烈建议）**：
   - 原始数据可能有500+列，计算极其耗时。
   - 点击 **"上传筛选列表"**，上传一个Excel文件，系统只取该文件**第一列**的有效数据作为需要分析的列名。系统会据此过滤无关列，大幅提升分析速度。

3. **开始分析**：
   - 点击 **“开始分析”** 按钮。根据数据量大小（10万条数据大约需十几秒至半分钟），请耐心等待。
   - 分析完成后，右侧将展示各类图表。

### 3.2 按时间维度深挖
如果发现某一段时间的不良率异常飙升，你可以：
1. 在左侧面板的 **“时间筛选”** 区域，设置起始和结束日期。
2. 选择你需要查看的 Top N（如 Top 5 或 Top 10）。
3. 点击 **“🔍 按时间筛选分析”**。
4. 图表会立刻更新，告诉你**在这段特定时间内**，导致NG的最核心要素是什么。

## 4. 图表解读指南

- **数据概览**：展示当前分析范围内的总样本数、Pass数、Fail数以及整体不良率。
- **相关性 Top N (条形图)**：排名越靠前、分数越高的特征，就是导致NG的“最大嫌疑人”。
- **Pass/NG 分层对比 (双色柱状图)**：展示核心嫌疑特征在好品和坏品中的分布情况。如果某个类别（比如2号喷嘴）的橙色柱子（Fail）异常高，说明问题很大。
- **硬件单元归因 (柱状图)**：专门提取了所有的机台、机头、喷嘴等硬件参数，直接告诉你哪个硬件产出的NG最多，并标出该硬件自身的NG率。
- **时序参数分布对比 (箱线图)**：用于分析连续型数据（如时间、温度）。可以直观看出好品和坏品在某个参数上的分布重心是否发生了“偏移”。
- **NG趋势 (折线柱状图)**：按天展示每天的总Fail数量和不良率波动。
- **失败工站分布 (饼图)**：直观展示不良品都集中拦截在哪些工站。

## 5. 常见问题
- **遇到“未找到Results列”报错**：请检查你的Excel文件，将表示良率结果的那一列的表头改名为 `Results`。
- **分析时间过长**：10万行数据进行上百次相关性矩阵计算属于CPU密集型任务。建议必须上传“筛选表”来减少计算维度，或者截取部分时间段的数据进行分析。
