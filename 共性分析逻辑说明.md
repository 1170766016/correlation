# 终检 Fail 产品共性聚集度分析逻辑说明 (企业交付级)

本文档详细说明了全制程不良归因分析工具中的核心算法架构。该算法经过优化，引入了严谨的统计学多重检验校正，确保分析结果在工业海量维度下的高可信度。

---

## 1. 核心统计体系

分析引擎通过 **“提升度 (Lift) + 显著性检验 (Significance Test) + FDR 校正”** 三位一体的逻辑，从数百个工序维度中筛选出真正的异常聚集点。

### 1.1 核心指标：Lift (提升度)
*   **公式**：`Lift = P(Fail | 特征取值) / P(Fail | 基准池)`
*   **基准池 (Baseline Pool)**：始终使用全量数据（**PASS + 全量 FAIL**）。即使在 UI 中筛选了特定的 `Failed_Station`，算法分母也会考虑全局频率，以防止因基准组偏差导致 Lift 被人为抬高。
*   **业务意义**：如果 Lift = 5，说明当该特征取值出现时，产品的 Fail 概率是整体平均水平的 5 倍。

### 1.2 混合显著性检验 (Hybrid Significance Test)
为了平衡计算效率与小样本下的准确性，引擎采用混合策略：
*   **大样本 (N > 1000 且 2x2 单元格均 > 5)**：采用 **卡方检验 (Chi-Squared Test)** 进行快速显著性评估。
*   **小样本**：采用 **Fisher 精确检验 (Fisher's Exact Test)**。这是工业制程分析中最严谨的方法，能够处理只有个位数 NG 样本的情况。

### 1.3 核心修复：BH-FDR 多重检验校正 (FDR-q)
这是区分“实验室代码”与“企业交付级代码”的关键。
*   **背景**：当你在分析 500 个工序特征（产生数千个假设）时，按传统的 `p < 0.05` 阈值，会有大量的假阳性（误报）。
*   **逻辑**：引擎引入了 **Benjamini-Hochberg (BH) 过程**。它会对所有候选结果的 p-value 进行升序排列并动态校正。
*   **FDR-q 值**：只有通过 FDR 校正后的显著性值（q-value）小于 0.05 的结果才会被展示。这意味着我们将整个排行榜的“虚假信号”比例控制在 5% 以内。

---

## 2. 特征工程逻辑

### 2.1 离散特征处理
*   **自动归一化**：将 `1.0` (浮点) 与 `1` (整型) 自动合并，避免因数据类型不一致导致的特征分裂。
*   **空值处理**：识别并过滤常见的无效空值（`NaN`, `None`, `N/A`, `''` 等）。

### 2.2 唯一性诊断 (Unique Value Filter)
*   如果某个特征在 Fail 产品中 **100% 集中**（唯一取值只有一个），程序会将其识别为“强信号”。即使该特征在基准中也很常见（导致 Lift 不高），也会在 UI 底部以 **⚠️ 重点关注** 区域独立展示，用于识别可能导致全量失效的工艺配置（如固定的治具 ID）。

### 2.3 日期时间小时段 (Hour) 派生
*   系统不仅按“天”分析，还会自动提取工序结束时间中的“小时” (`MM-DD HH:00`)。
*   这有助于识别与**排班换岗、白夜班环境温湿度波动、设备预热**相关的周期性异常。

## 3. UI 综合评分逻辑

系统最终的排行榜并非仅看 Lift，而是采用加权 **综合评分 (Composite Score)**：
`综合评分 = Lift * W1 + Fail内占比 * W2`

*   **Lift**：代表“关联强度”（是否异常）。
*   **Fail 内占比**：代表“覆盖面”（是否能解释大部分 NG 样本）。
*   **默认权重**：推荐 Lift 权重设为 1.0。在小样本场景下，可以适当降低 Lift 权重，调高占比权重。

---

## 4. 统计学严谨性保证清单

| 风险点 | 我们的处理逻辑 |
| :--- | :--- |
| **假阳性 (误报)** | 强制通过 BH-FDR 校正，通过 q-value 过滤。 |
| **小样本陷阱** | 至少满足 `fail_count >= 3` 且通过 Fisher 精确检验。 |
| **基准偏差** | 基准池强制包含全量样本，不随 Failed_Station 筛选而缩小。 |
| **数据稀疏性** | 自动过滤唯一值过多（> 30% 记录数）的列（如 UUID）。 |
