# 🛠️ 部署与维护指南

本指南用于指导如何将“终检 Fail 产品共性聚集度分析平台”部署到正式环境或个人工作站。

## 1. 环境准备

### 1.1 Python 环境
推荐使用 **Python 3.10+**。

### 1.2 安装依赖
在项目根目录下，使用 pip 安装所有必要库：
```bash
pip install -r requirements.txt
```
*主要依赖包括：`streamlit`, `pandas`, `plotly`, `requests`, `openpyxl`。*

---

## 2. 配置与优化

### 2.1 配置大模型 (LLM)
如果您公司有自建的 LLM 接口，请在 `app.py` 顶部修改以下变量：
*   `LLM_API_URL`: 填写 API Endpoint。
*   `LLM_MODEL`: 填写对应的模型名称。

### 2.2 性能调优
针对每日 40 万行的大数据量，请务必参考 `性能优化与列配置指南.md`，将您关注的 200 个列名配置到白名单中。

---

## 3. 启动应用

### 3.1 本地测试启动
```bash
streamlit run app.py
```

### 3.2 后台长期运行 (Linux 服务器)
如果您在 Linux 服务器上部署，建议使用 `nohup` 让程序在后台持续运行：
```bash
nohup streamlit run app.py --server.port 8501 > streamlit.log 2>&1 &
```

---

## 4. 常见问题排查 (FAQ)

### Q1: 页面上传大文件后崩溃？
*   **原因**：服务器内存（RAM）不足。
*   **对策**：开启 `usecols` 优化（详见优化指南），或将 `.xlsx` 转换为 `.csv` 后再上传。

### Q2: 无法从其他电脑访问？
*   **对策**：启动时指定监听地址：
    `streamlit run app.py --server.address 0.0.0.0`
    并确保防火墙已开放 8501 端口。

### Q3: LLM 总结很慢或超时？
*   **原因**：网络延迟或模型响应慢。
*   **对策**：代码中默认超时为 60s，如需调整可修改 `requests.post` 中的 `timeout` 参数。

---
**维护建议**：
建议每周清理一次 `streamlit.log` 日志文件，并根据生产数据的变化更新 `app.py` 中的列名白名单。
