# 时间日期类 Fail 聚集度计算说明

## 一、计算流程

### 1. 时间列识别

自动识别数据表中所有以 `_End_Time`、`_Start_Time`、`_datetime` 结尾的列（不区分大小写），
这些列的值是时间戳，例如 `2026-04-13 13:24:45` 或 `2026/04/13 13:42:51`。

> `_Staging_time`（持续秒数，如 `3`、`45`、`82`）被排除，不参与时间分析。

### 2. 日期+小时特征提取

对每个时间列，用 `pd.to_datetime` 解析时间戳，然后提取 `月-日 时:00` 格式：

```
原始值: 2026-04-13 13:24:45
  → 提取: 04-13 13:00  （丢弃分钟和秒，按小时聚合）

原始值: 2026/04/13 21:08:31
  → 提取: 04-13 21:00
```

新列命名为 `{原列名}_DayHour`，例如 `VCM_M1_FPC_UpCoil_Attach_End_Time_DayHour`。

### 3. 有效值过滤

有效解析率低于 30% 的列跳过（防止全是脏数据的时间列参与计算）。

### 4. Fail 占比计算

对每个 `_DayHour` 列，与普通离散特征共用同一套 `compute_lift` 逻辑，计算每个（工序, 日期+小时）的 Fail 内占比：

```
fail_ratio = 该(工序,时段)的Fail数 / 全部Fail数
```

例如：`VCM_M8_Aging 在 04-06 02:00 有 17 个 Fail / 全部 150 个 Fail = 11.3%`

### 5. TOP 10 展示

- 过滤保留 `fail_ratio > 5%` 的条目
- 按 fail_ratio 降序取全局 TOP 10（不同工序混合排名）
- 横向柱状图展示，左侧标签为 `工序名 · 时间`

## 二、实际业务价值

### 场景 1：排查时段性异常

| 发现 | 可能原因 |
|------|---------|
| `VCM_M8_Aging` 在 `04-06 02:00` 占比 11% | 该时段 Aging 设备故障 / 操作员交接班 |
| `VCM_Function_Test1` 在 `04-16 15:00` 占比 10% | 下午班开始时测试参数未校准 |
| 多个工序在同一日期同一小时同时异常 | 该时段存在全流程共性异常（如温湿度、物料批次） |

### 场景 2：区分工序 vs 时段问题

- 如果高占比集中在**特定工序 + 特定时段** → 该工序在该时段存在异常（设备、人员）
- 如果高占比集中在**特定时段 + 多道工序** → 可能是环境因素或来料问题
- 如果高占比集中在**特定工序 + 全时段** → 该工序本身存在系统性问题（非时间相关性）

### 场景 3：排班/调机追溯

结合生产排班表，定位异常时段对应的操作员、调机记录、换线事件，
缩小排查范围。

---

## 三、与普通离散特征分析的区别

| 维度 | 普通离散特征（MC_ID/Cavity等） | 时间日期特征 |
|------|-------------------------------|-------------|
| 特征粒度 | 设备/模穴/批次/供应商 | 每道工序的日期+小时 |
| 分析目标 | 哪个设备/来料有问题 | 哪道工序在什么时段有问题 |
| 排序方式 | 综合评分(Lift+占比) | Fail内占比 |
| 过滤阈值 | fail_ratio > 20% | fail_ratio > 5%（小时分散在 24 小时，占比天然较低） |
