# 🛠️ 项目适配与修改指南

当你从公司拿到真实的 Excel 数据后，可能会发现列名（如日期不叫 `Date`）、判定内容（如结果不叫 `Pass/Fail`）与 Demo 不同。请按照以下步骤快速修改代码。

---

## 1. 核心列名映射 (Column Mapping)

在 `app.py` 中，程序依赖几个关键列进行统计。如果你的 Excel 表头不同，请在 `app.py` 中搜索并修改以下位置：

### 判定结果列 (`Results`)
- **默认名称**: `Results`
- **修改位置**: `app.py` 第 253-262 行。
- **说明**: 程序会自动尝试匹配 `NG, Fail, FAIL` 等关键词。如果你的判定列叫“结果”或“判定”，程序可能找不到。
- **强制指定**: 你可以在第 253 行前直接指定：
  ```python
  # 强制指定 '判定结果' 列为 Results
  if '判定结果' in df.columns:
      df = df.rename(columns={'判定结果': 'Results'})
  ```

### 日期时间列 (`Date`)
- **默认名称**: `Date`
- **修改位置**: `app.py` 第 152, 171, 221, 286 行。
- **说明**: 用于 NG 趋势图和时间过滤。如果你的列叫 `InspectTime` 或 `日期`，请全文替换 `'Date'` 为你的列名。

### 工站与不良模式
- **工站列**: `Failed_Station` (第 138, 221 行)
- **模式列**: `Failure_Mode` (第 145, 221 行)
- **说明**: 如果没有这两列，相关图表（饼图）将不显示。

---

## 2. 判定结果标准化 (Normalization)

公司数据可能是 `OK/NG`, `0/1`, `良品/不良品`。

- **修改位置**: `app.py` 第 265-266 行。
- **代码参考**:
  ```python
  # 原代码：将 fail, ng, failed 识别为 'Fail'，其余为 'Pass'
  df['Results'] = df['Results'].astype(str).apply(
      lambda x: 'Fail' if x.lower() in ['fail','ng','failed','不良','1'] else 'Pass')
  ```
- **注意**: 程序内部逻辑（如绘图、计算）统一使用 `'Fail'` 和 `'Pass'` 作为 key，所以只需在这里把你的原始数据映射过去即可。

---

## 3. 相关性分析排除列表 (Skip List)

有些列（如 SN、ID、时间戳）具有唯一性或无统计意义，**必须排除**在相关性计算之外，否则会产生虚假的高分。

- **修改位置**: `app.py` 第 48-49 行的 `skip` 集合。
- **操作**: 将你不希望参与计算的列名（如 `Worker_ID`, `Barcode` 等）加入这个集合。

---

## 4. 硬件归归因关键词 (Hardware Keywords)

硬件归因图表会自动寻找包含特定关键词的列（如吸嘴号、模具号）。

- **修改位置**: `app.py` 第 104 行。
- **代码参考**:
  ```python
  # 增加你公司特有的硬件标识，如 'Cavity', 'Mold', 'Carrier'
  hw_keywords = ('Nozzle', 'Socket', 'MC_ID', 'Head_ID', 'Cavity', '治具')
  ```

---

## 5. 日期格式处理

如果 `pd.to_datetime` 无法自动识别你的日期格式（例如是 `20240507` 这种纯数字格式）：

- **修改位置**: `app.py` 第 153, 172, 287 行。
- **代码参考**:
  ```python
  # 显式指定格式
  date_series = pd.to_datetime(df['Date'], format='%Y%m%d', errors='coerce').dt.date
  ```

---

## 6. 内存与性能调整

如果你的数据超过 50 万行，Excel 加载会变慢。

- **文件大小限制**: `app.py` 第 13 行 `MAX_CONTENT_LENGTH` 默认 500MB，可根据需要调大。
- **列过滤**: 建议使用**“上传筛选列表”**功能。在小 Excel 中列出你关心的 50-100 个列名上传，程序会只加载这些列，速度提升 5-10 倍。

---

## 💡 修改建议流程
1. **先改 `Results`**: 确保程序能分清什么是“好”什么是“坏”。
2. **再改 `Date`**: 确保时间轴能跑通。
3. **更新 `skip` 列表**: 剔除无意义的列（如 ID、备注等）。
4. **运行测试**: 只要 `api/analyze` 接口不报 `500` 错误，前端图表就会自动适配显示。
