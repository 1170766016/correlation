# 统计分析与相关性计算原理说明

这份文档详细阐述了本系统用于排查“全制程NG不良归因”的数学原理，以及各个图表背后的计算逻辑。当你需要向老板或同事解释“为什么系统认定某个因素是导致不良的主因”时，可以参考本文档。

---

## 1. 为什么不直接用 R²？

在沟通中，你同事提到了使用 **R² (决定系数)**。但在实际的工业制程分析中，针对我们的数据结构，直接使用 R² 并不是最科学的选择，原因如下：

- **R² 适用场景**：通常用于**线性回归模型**中，评估一个或多个连续型自变量（如温度、压力）对一个**连续型因变量**（如良率百分比、重量偏差）的解释程度。
- **我们的场景**：我们的目标变量（结果列 `Results`）是 **Pass / Fail (合格/不良)**，这是一个标准的**二分类离散变量**。
- **解决方案**：为了最准确地衡量各个特征与“是否NG”的相关性，系统在后台实现了一套**自适应分流算法**：
  - 如果分析的列是**分类变量**（如：机台号 `MC_ID`、喷嘴号 `Nozzle`、供应商 `Vendor`），系统采用 **Cramér's V (克莱姆 V 系数)**。
  - 如果分析的列是**连续变量**（如：驻留时间 `Staging_time`），系统采用 **点双列相关系数 (Point-biserial Correlation)**。

---

## 2. 核心相关性算法原理

### 2.1 分类变量：Cramér's V (克莱姆 V 系数)
**应用对象**：机台号、测试Socket、喷嘴编号、物料批次号等。
**计算原理**：
1. 系统首先提取某一列（比如 `VCM_M8_Aging_Nozzle`），找出它包含的所有分类（如 Nozzle_1, Nozzle_2...）。
2. 构建一个交叉表（列联表），统计每个喷嘴产出的 Pass 数量和 Fail 数量。
3. 利用 **Pearson 卡方检验 (Chi-square Test)** 计算出观测频数与期望频数之间的偏离程度（卡方值 $\chi^2$）。
4. 将卡方值标准化，计算出 Cramér's V 值。
   - **公式**：$V = \sqrt{\frac{\chi^2}{n \times \min(r-1, c-1)}}$
   - $n$ 是总样本数，$r$ 和 $c$ 是交叉表的行数和列数。
**如何解读**：V 值的范围是 0 到 1。数值越接近 1，说明该硬件/批次与 NG 结果的关联度越极度强；越接近 0，说明它是随机分布的，不背锅。

### 2.2 连续变量：点双列相关系数 (Point-biserial)
**应用对象**：各种耗时参数、温度、时间戳差值（如 `Staging_time`）。
**计算原理**：
1. 专门用于衡量一个二元分类变量（Pass=0, Fail=1）和一个连续型变量之间的线性相关程度。
2. 实质上是 Pearson 积矩相关系数在特定条件下的简化版。
3. **公式**：$r_{pb} = \frac{M_1 - M_0}{s_n} \sqrt{\frac{n_1 n_0}{n^2}}$
   - $M_1, M_0$ 分别是 Fail 组和 Pass 组在该参数上的平均值。
   - $s_n$ 是总体标准差，$n_1, n_0$ 是两组的样本数。
**如何解读**：系统最终取该系数的**绝对值**（0 到 1）。数值越大，说明这个参数的波动越容易导致产品变成 Fail。

## 3. 图表可视化原理

### 📊 图表 1：相关性 Top N (Correlation Score Ranking)
- **原理**：系统遍历所有特征列（若上传了筛选列表则仅遍历筛选后的列），分别用上述两种算法算出相关性得分（统一映射到 0~1 的区间）。
- **展示**：从高到低排序，切片取出 Top 5 或 Top 10。排在第一的就是本次分析范围内的“头号嫌疑人”。

### 📈 图表 2：Pass/NG 分层对比 (Stratified Comparison)
- **原理**：针对被揪出来的“头号嫌疑人”（通常是分类变量，如某个工站的机台分布），系统将其数据强行“分层（Stratified）”。
- **计算逻辑**：对于该特征下的每一个类别（如 MC_A, MC_B, MC_C），分别统计它们对应的 Pass 品数量（青色柱）和 Fail 品数量（橙色柱）。
- **业务意义**：如果某个机台的橙色柱体异常高，而青色柱体相对较矮，这就构成了该机台是“偏移”不良主因的铁证（实锤）。

### 🔧 图表 3：硬件单元归因 (Hardware Unit NG Attribution)
- **原理**：专门针对表头中包含 `MC_ID`（机台）、`Nozzle`（喷嘴）、`Socket`（穴位）、`Head_ID`（机头）的硬件类字段进行穷举扫描。
- **计算逻辑**：不看相关性，只看绝对的 NG 数量。找出在 Fail 品中出现频次最高的具体硬件（如 `VCM_M8_Aging_Nozzle = Nozzle_2`）。
- **复合指标**：图表柱体高度代表 **NG绝对数量**，上方标注的百分比代表**该硬件的自身不良率**（即：用它生产的全部产品中，有多少报废了）。

### 📦 图表 4：时序参数分布对比 (Box Plot: Pass vs NG)
- **原理**：针对被评为高相关性的“连续型变量”（如某段烘烤停留时间），采用箱线图（Boxplot）对比好坏品的分布差异。
- **计算逻辑**：
  - 分别提取 Pass 组和 Fail 组在该时间参数上的全部数据。
  - 计算统计学五数概括：**最小值、第一四分位数(Q1)、中位数、第三四分位数(Q3)、最大值**。
- **业务意义**：如果 Pass 组的箱体和 Fail 组的箱体在垂直方向上出现了明显的“错位”（偏移），说明这个时间参数设置不当（偏高或偏低）是导致 NG 的直接物理原因。

## 4. 时间切片（防聚集效应）
- **时间筛选的逻辑**：由于工业生产中经常出现“某批物料在某个时间段内集中爆发不良”，全生命周期的统计往往会稀释这种波动。
- **实现原理**：通过前端传入的时间范围（如 `start_date` 到 `end_date`），系统会在 Pandas 后端通过掩码（Mask）对 DataFrame 进行截断，然后**仅在选定的时间切片内，重新运行上述所有的相关性计算与特征排序**。
- 这正是找出“这段时间关键要素是什么”的核心机制。
